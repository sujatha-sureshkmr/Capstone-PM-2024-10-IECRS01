Use Case: Funds Transfer Scenario

The bank's objective is to ensure seamless assistance for customers seeking to transfer payments or funds to their registered recipients. Upon the bank's receipt of the transfer request at the backend, a prompt notification will be sent to the customer, acknowledging the initiation of the transfer process. Once the transfer undergoes processing and reaches completion, a second notification will be promptly dispatched to the customer. This notification will comprehensively detail the outcome of the processing, providing essential information regarding the successful transfer or any potential issues encountered during the transaction. This approach aims to maintain transparency and provide customers with real-time updates throughout the transfer procedure, ensuring a smooth and reliable banking experience.

Funds Transfer Scenario - Sequence of Events

  1. User triggers a transfer of funds from their mobile banking
  2. Backend for Frontend service invokes call to backend systems invokes the API
  3. API invokes persistent queue on the queue manager
  4. The queue manager is configured to work with streaming queue which will post the message to the events for processing by the subscribed event consumers
  5. Events are processed on the event brokers
  6. Event consumers process the inbound event
  7. Core banking processes the transfer
  8. When processing is complete core banking triggers notification to the events to be picked up by the event consumers
